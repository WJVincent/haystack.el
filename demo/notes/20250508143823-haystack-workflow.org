#+TITLE: Haystack Workflow
#+DATE: 2025-05-08
# %%% haystack-end-frontmatter %%%
A typical Haystack workflow consists of three recurring loops: capture (write
new notes quickly), search (find relevant notes by content), and explore
(navigate related notes through tree view or MOCs). The capture loop begins with
~haystack-new-note~: type a title, press enter, write the note body, and save —
the entire sequence takes under 30 seconds for a typical atomic zettel. The
search loop is the core daily workflow: a query via ~haystack-run-root-search~
returns a grep-mode results buffer, and pressing ~RET~ on any match opens that
file at the matching line. For the exploration loop, ~haystack-copy-moc~ or
~haystack-show-tree~ from a results buffer expands outward to reveal
related notes you may have forgotten existed but are highly relevant to your
current thinking. The weekly review loop (not a Haystack-specific concept but
important to any second-brain) involves opening recently created notes,
processing captures from daily notes, and updating MOCs with newly relevant
zettel. Integration with org-capture enhances the capture loop: a capture
template can call ~haystack-new-note~ with a pre-filled title, making capture
from any buffer as fast as ~C-c c n title RET~. The search loop benefits from
progressive filtering: pressing =f= in a results buffer runs a new rg search
against only the files in the current results, creating a narrowed child buffer.
Each filter creates a child buffer in the search
tree, which you can navigate with =u= (up) and =d= (down). For writing projects,
a Haystack workflow might involve
opening an MOC for the topic, reviewing its linked notes, pulling
insights into a new synthesis note, and then linking back to the sources. The
entire workflow operates within Emacs without switching applications, making the
second-brain a genuinely integrated part of your coding, writing, and research
environment.
