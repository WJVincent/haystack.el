#+TITLE: Haystack Filtering
#+DATE: 2025-03-21
# %%% haystack-end-frontmatter %%%
Haystack filtering narrows the search space progressively through a series of
rg invocations, each scoped to the file set from the previous step. The
~/ filter~ syntax in Haystack restricts matches to notes
whose filename contains the given string: ~/emacs~ limits results to notes with
"emacs" in their filename, which in a flat zettelkasten corresponds to topic-
prefixed note names. File extension filtering is configured via ~haystack-file-glob~: set it to
~"*.org"~ to restrict searches to org files, or leave it nil to search all file
types in the notes directory. Date range filtering uses ~haystack-search-date-
range~, which searches for ~hs:~ timestamps within a user-supplied date range,
supporting year, month, day, or minute precision. Scope prefixes narrow which
part of each note is searched: =>= for body-only (after the frontmatter sentinel)
and =<= for frontmatter-only (before the sentinel). Filtering and expansion groups
compose: a filtered search can still expand synonyms, so you can search for
"pkm" within org files dated this year and have it also match "zettelkasten" and
"second-brain." Progressive filtering extracts the file set from the current results buffer and
runs a new rg invocation against only those files, reducing the corpus for each
successive child search. Haystack's progressive
filtering allows incremental refinement: from any results buffer, press =f= to
filter further, narrowing to files that also contain the new term. Each filter
creates a child buffer, building a tree of increasingly focused results. For
notes organized with naming conventions (e.g.,
~YYYYMMDD-topic-subtopic.org~), the ~/ filter~ against filename provides a
lightweight form of hierarchical navigation without needing actual
subdirectories. Filtering in Haystack is additive and composable — multiple
filter conditions narrow the corpus progressively, letting you home in on the
exact subset of notes relevant to your current task.
