Scheme is a minimalist Lisp dialect designed in the 1970s by Guy Steele and
Gerald Sussman, famous for its clean semantics, mandatory tail call
optimization, and use as a teaching language. The Scheme philosophy is radical
minimalism: the language core is kept as small as possible, with most
functionality derived from a handful of fundamental constructs. Scheme
standardizes through a series of "Revised Reports" (R5RS, R6RS, R7RS); R7RS-
small is the current widely-implemented standard, with a deliberate separation
between the small core and a larger library spec. Unlike Common Lisp (cl),
Scheme has a unified namespace for functions and variables — there is no
distinction between the "function cell" and "value cell" of a symbol, making it
a Lisp-1. First-class continuations via `call-with-current-continuation`
(call/cc) are unique to Scheme among mainstream languages; they allow capturing
the current execution state as a value and resuming it later. Hygienic macros
via `syntax-rules` and `syntax-case` prevent the variable capture problems of
traditional Lisp macros, introducing new bindings that cannot accidentally
shadow caller-scope names. Tail call optimization (TCO) is guaranteed by the
R5RS standard: any Scheme implementation must support arbitrarily deep tail
recursion without stack overflow. The `define` form in Scheme handles both
variable binding and function definition: `(define (square x) (* x x))` is sugar
for `(define square (lambda (x) (* x x)))`. Racket, a Scheme descendant, extends
the language substantially and is used in production software, PLT research, and
as the platform for the "How to Design Programs" curriculum. Scheme's influence
on emacs-lisp is significant: many of the newer emacs-lisp features like lexical
binding and proper closures were inspired by Scheme's cleaner scoping model.
