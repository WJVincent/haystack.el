#+TITLE: Haystack Configuration
#+DATE: 2025-04-01
# %%% haystack-end-frontmatter %%%
Haystack is configured through a small set of customizable variables, all
following the ~haystack-~ prefix convention for emacs-lisp packages. The most
important variable is ~haystack-notes-directory~: a string path to the directory
containing your notes, which all search and creation commands use as their root.
Expansion groups are stored in a ~.expansion-groups.el~ file in the notes
directory — an alist of synonym groups that Haystack expands into alternation
patterns when constructing rg queries. ~haystack-default-extension~ controls the
default file extension for new notes (e.g., ~"org"~ or ~"md"~).
~haystack-file-glob~ restricts which files are searched (e.g., ~"*.org"~); nil
means search all files. ~haystack-context-width~ controls how many characters
of context surround each match in results buffers. Frecency data is stored
automatically in ~.haystack-frecency.el~ within the notes directory.
~haystack-frontmatter-functions~ is an alist mapping file extensions to
frontmatter-generating functions, making the format extensible to any file type.
The ~use-package~ configuration block for Haystack pairs naturally with ~:init~
for binding the prefix map and ~:custom~ for the directory and display settings.
A minimal working Haystack configuration requires only ~haystack-notes-directory~
set to a valid path; all other variables have sensible defaults that work well
for most users without adjustment.
