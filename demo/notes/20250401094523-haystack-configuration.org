#+TITLE: Haystack Configuration
#+DATE: 2025-04-01
# %%% haystack-end-frontmatter %%%
Haystack is configured through a small set of customizable variables, all
following the ~haystack-~ prefix convention for emacs-lisp packages. The most
important variable is ~haystack-notes-directory~: a string path to the directory
containing your notes, which all search and creation commands use as their root.
~haystack-expansion-groups~ is the second key configuration: a list of synonym
groups, each a list of strings, that Haystack expands into alternation patterns
when constructing rg queries. ~haystack-frecency-file~ specifies where Haystack
stores the visit frequency and recency data; defaults to a file within ~user-
emacs-directory~ but can be redirected to your notes directory for version
control. ~haystack-search-extensions~ controls which file types are included in
search: defaults to org and Markdown, but can be extended to include ~.md~,
~.txt~, ~.el~, ~.py~, or any plain-text format. ~haystack-new-note-format~
determines the default format for new notes: ~'org~ generates org frontmatter,
~'markdown~ generates YAML frontmatter, or a custom template string can be
provided. The ~use-package~ configuration block for Haystack pairs naturally
with ~:bind~ for keybinding setup, ~:custom~ for the directory and expansion
groups, and ~:hook~ for any mode activations. ~haystack-rg-extra-args~ allows
passing additional flags to ripgrep (rg): for example ~"--type-add
'note:*.org'"~ to define a custom file type, or ~"--max-depth 1"~ for a shallow
search. ~haystack-result-display-function~ can be customized to change how
selected notes are opened — in the current window, in an other-window, or in a
side panel. A minimal working Haystack configuration requires only ~haystack-
notes-directory~ set to a valid path; all other variables have sensible defaults
that work well for most users without adjustment.
