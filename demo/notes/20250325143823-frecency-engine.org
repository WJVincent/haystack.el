#+TITLE: Haystack Frecency Engine
#+DATE: 2025-03-25
# %%% haystack-end-frontmatter %%%
Frecency is a portmanteau of "frequency" and "recency." Haystack's frecency
engine tracks /search chains/, not individual notes: the chain =rust → bevy →
ecs= is the unit that gains score each time you run it. Score is a simple
recency-weighted count — visits divided by days since last access — so chains
you use often and recently float to the top while old chains gradually decay.

The frecency data is stored in =.haystack-frecency.el= as an association list
mapping chain keys to visit records. Each chain key is a descriptor-shaped
plist recording the root term, any filters, and their modifiers. The file is
updated on idle or shutdown, not on every search, keeping interaction fast.

Frecency addresses the core limitation of text search: in a large corpus, many
chains may be valid, and text relevance alone cannot distinguish "the chain I
usually run" from "a chain that happens to work." When you call
=haystack-frecent=, only /leaf/ chains appear by default — chains where no
deeper extension has a higher score. This keeps the list focused on your actual
exploration endpoints rather than intermediate steps.

The engine adapts to changing work patterns: if you spent last month on
emacs-lisp and this month shift to pkm topics, the ranking automatically
reflects your current focus within a few sessions. Haystack's frecency model
is inspired by the Firefox URL bar's frecency algorithm and by z/zoxide's
directory navigation frecency, both proven effective for recall-heavy
interfaces.
