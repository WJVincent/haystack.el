#+TITLE: Haystack Tree View
#+DATE: 2025-03-27
# %%% haystack-end-frontmatter %%%
Haystack's tree view displays the forest of open search and filter buffers as
an indented outline in a dedicated =*haystack-tree*= buffer, providing a map of
your active search sessions. The ~haystack-show-tree~ command (bound to =t= in
results buffers and =C-c h t= in the prefix map) opens this view. Each node in
the tree represents a results buffer: root searches appear at column zero, and
each progressive filter is indented one level deeper beneath the buffer it was
filtered from. The current buffer is marked with =←=. Multiple disconnected
search sessions appear as separate trees separated by blank lines. Pressing
=RET= on a node visits that results buffer; =q= closes the tree window. The
tree derives from =special-mode= and provides =n=/=p= for item navigation and
=M-n=/=M-p= for sibling navigation within the same depth level. Each depth
level gets a distinct face drawn from the active theme via
~haystack-tree-depth-faces~. When navigating to children from a results buffer
with one child, Haystack goes directly; when multiple children exist, a
=*haystack-children*= picker lists them. The tree view complements search
rather than replacing it: use search to find what you need, then use the tree
to retrace your search path and see how your filters branched. The tree is
purely navigational — it reflects which buffers were filtered from which, not
any relationship between note contents. It does not persist or affect the
underlying files.
