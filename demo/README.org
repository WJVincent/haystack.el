#+TITLE: Haystack Demo Corpus
#+STARTUP: overview

* Overview

This directory contains a pre-built corpus of 84 sample notes for
trying out Haystack without touching your own notes.

Run ~M-x haystack-demo~ to activate demo mode.  Haystack will copy
these notes into a temporary directory and point itself there.  A
warning banner appears in every results buffer.  When you are done,
~M-x haystack-demo-stop~ kills all demo buffers, deletes the temp
directory, and restores your previous ~haystack-notes-directory~.

* Corpus layout

The 110 notes span six topic areas and include deliberately messy
files to demonstrate that Haystack handles real-world imperfection:

| Topic      | Notes | File types                    |
|------------+-------+-------------------------------|
| Emacs      |    18 | .org                          |
| Lisp       |    20 | .org, .md                     |
| PKM        |    19 | .org, .md                     |
| Haystack   |    19 | .org, .md                     |
| Code       |    13 | .el .py .js .lua .rb .rs      |
| Humanities |     8 | .org, .md, .txt               |
| Messy      |    13 | .org, .md, .txt (some no frontmatter, spaces in names) |

* Pre-built expansion groups

The following synonym groups are pre-configured in
~.expansion-groups.el~.  Any member term expands to the full
alternation during search:

| Root    | Synonyms                                          |
|---------+---------------------------------------------------|
| lisp    | common-lisp, cl, scheme, clojure                  |
| elisp   | emacs-lisp, emacs lisp                            |
| pkm     | zettelkasten, knowledge-management, second-brain  |
| search  | ripgrep, rg                                       |
| note    | notes, zettel                                     |
| writing | composition, essay, draft                         |

* Frecency history

The ~.haystack-frecency.el~ file ships empty so you can experience
frecency from scratch.  After exploring the corpus with a few
searches, run ~M-x haystack-demo-seed-frecency~ to populate it with a
realistic pre-built search history.

Once seeded, run ~C-c h f~ (~haystack-frecent~) to replay frequent
searches.  By default only *leaf* entries are shown — chains that are
not merely an intermediate step toward a more-visited deeper search.
Use ~C-u C-c h f~ to see all recorded chains, including intermediate
ones dominated by their deeper siblings.

* Suggested walkthrough

** 1 · Root search
   ~C-c h s~ → type =emacs= and press RET.  Note the expanded query
   in the header: =emacs= is not in an expansion group, so it matches
   literally.  Then try =lisp=: the header shows the full alternation
   =(lisp|common-lisp|cl|scheme|clojure)= because "lisp" has an
   expansion group.

** 2 · Progressive filtering
   From the =lisp= results buffer, press =f= and type =macros=.  You
   now have a child buffer scoped to files that matched =lisp= *and*
   contain =macros=.  Press =u= to go back up.  Press =f= again and
   try =!macros= (negation) to see files about lisp that do *not*
   mention macros.

** 3 · Frecency: before and after
   Start by running ~C-c h f~ — the list is empty because you haven't
   searched yet.  Do a few searches (=emacs=, =lisp=, =pkm=) and
   filter some of them.  Now ~C-c h f~ shows your actual history.

   To see a richer history, run ~M-x haystack-demo-seed-frecency~.
   Now ~C-c h f~ shows only leaf searches.  Notice that broad terms
   like =emacs= and =haystack= are hidden — they are dominated by
   deeper, more-visited chains like =emacs > org-mode= and
   =haystack > filtering=.  Use ~C-u C-c h f~ to reveal all chains,
   or open ~M-x haystack-describe-frecent~ and press =v= to toggle
   the view.

** 4 · MOC generator
   From any results buffer, press =c= to copy a Map of Contents to
   the kill ring, then ~C-c h y~ to yank it into any note.  Open one
   of the code files and yank there to see language-appropriate
   formatting: Python gets a list of dicts, Lua gets a local table,
   Ruby and Rust fall back to comment style (no built-in data
   formatter).

** 5 · Create a note
   ~C-c h n~ → enter a slug → choose an extension.  The new note
   will carry a "HAYSTACK DEMO NOTE" label in its body as a reminder
   that it lives in the temporary directory and will be discarded on
   ~haystack-demo-stop~.

** 6 · Discoverability
   Open one of the orphan notes (filter =/orphan= from any results
   buffer).  With the note buffer active, run ~M-x
   haystack-describe-discoverability~.  A report will open showing
   every unique token in the note, grouped into tiers:
   - *Isolated* — terms that appear in no other note (vocabulary gap)
   - *Sparse* — terms that appear in 1–2 other notes
   - *Connected* — terms shared with several notes
   - *Ubiquitous* — terms that appear almost everywhere

   Try it on =orphan-vocabulary-gap= — its Isolated tier is large,
   confirming that the note's vocabulary is genuinely unique.

** 7 · Messy files
   The corpus includes deliberately imperfect files: notes without
   frontmatter, files with spaces in their names, scratch notes
   without timestamps, and a long conference brain dump.

   Try ~C-c h s~ → =sourdough= to find the recipe.  Or filter
   =/reading list= (filename filter with a space) to test that
   Haystack handles spaces in filenames.  Search =TODO= to see how
   rough drafts and task lists surface alongside polished notes.

** 8 · Cross-domain connections
   The corpus spans technical and humanistic topics.  Search
   =synthesis= to see hits from PKM notes, 19th-century novel
   analysis, and film editing theory.  Search =atomicity= to find
   both the note-taking principle and the Stoic philosophical
   parallel.  Search =navigation= to see urban planning alongside
   Emacs buffer management.

   Try =writing= — it expands to =(writing|composition|essay|draft)=
   via the expansion group, pulling in results from film editing,
   music theory, and literature notes.

** 9 · Compose
   From any results buffer, press =f= and type =/synthesis=.  You'll
   land on four notes that bridge two topic areas:
   - =synthesis-emacs-pkm= (Emacs ∩ PKM)
   - =synthesis-org-roam-vs-haystack= (PKM ∩ Haystack)
   - =synthesis-plain-text-pkm= (PKM ∩ Haystack)
   - =synthesis-haystack-pkm= (Haystack ∩ PKM)

   Press ~C-c C-c~ (~haystack-compose~) to open a staging buffer.
   Each matched file appears as a section with context.  Edit freely,
   then ~C-c C-c~ again to commit the composite note to disk.

* Stopping the demo

~M-x haystack-demo-stop~

This kills all Haystack results buffers, kills any file-visiting
buffers inside the temp directory, deletes the temp directory, and
restores your previous ~haystack-notes-directory~.
