#+TITLE: Haystack — Development Roadmap
#+STARTUP: overview

* Phase 1 — MVP: The Core Loop
Create notes, search them progressively, export links. Ship this and use it daily.

** Definition of Done
- [X] Can create a note with timestamped slug, frontmatter, and sentinel
- [X] Can run a root search and get a grep-mode buffer with header
- [X] Can filter that buffer (positive and negated) and get child buffers
- [X] Can navigate the buffer tree (up, kill node, kill subtree, kill all, kill orphans)
- [X] Can yank a MOC from any results buffer
- [X] ~compile-goto-error~ works from every results buffer (implicit via grep-mode format)
- [X] Vertico/Orderless buffer switching works with the naming scheme (free from =*haystack:N:term*= convention)
- [X] Tree view and header navigation buttons complete
- [ ] Manually tested against a 100+ note corpus

** Input Processing Pipeline
*** DONE Prefix detection and stripping
- =!= (negate — filter only), ~=~ (literal bypass), =~= (raw regex)
- Composition: =!=term=, =!~pattern=, ~=~pattern~
- Strip in order, pass flags downstream
*** DONE Single-word vs multi-word classification
- Whitespace boundary after trimming
- Hyphenated/dotted terms are single-word
*** DONE Regex escaping layer
- Default: =regexp-quote= on bare term
- Skip if =~= prefix present
- Phase 2 expansion hook point noted in =haystack--build-pattern=
- All implemented via =haystack--parse-input= pipeline

** Progressive Filter — Root Search
*** DONE =haystack-run-root-search=
- Run input through prefix detection → classification → escaping
- In Phase 1: no expansion groups, all single-word terms are literal
- =rg -n -i= with directory scope
- Apply =haystack-file-glob= if set
- Apply composite filter glob (=--glob '!@*'= by default)
- =composite-filter= parameter (='exclude= / ='only= / ='all=) wired through for Phase 2 transient
*** DONE Spawn results buffer
- Name: =*haystack:1:TERM*=
- Insert header: =;;; haystack: root=TERM | N files, M matches=
- Header lines get =read-only= text property
- Set buffer-local =haystack--search-descriptor= and =haystack--parent-buffer= (nil)
- =switch-to-buffer= for filter-further (preserves window layout); =pop-to-buffer= for root search
*** DONE =haystack-search-region=
- Take active region, feed into =haystack-run-root-search=
*** DONE Result statistics from rg output
- File count = unique filenames in output
- Match count = total output lines minus header lines
*** DONE Result line truncation
- Content truncated to =haystack-context-width= chars (default 60) around the match
- =...= ellipsis at truncated ends; =file:line:= prefix preserved for =compile-goto-error=
- =haystack-context-width= defcustom for user tuning

** Progressive Filter — Child Filters
*** DONE =haystack-filter-further=
- Prompt for filter term, run through input pipeline
- Extract unique filenames from current buffer (skip header lines)
- Write filenames to temp file via =haystack--write-filelist=
- Run =haystack--xargs-rg= with new term scoped to those files
- Delete temp file after search (=unwind-protect=)
*** DONE Negation filter (=!= prefix)
- Step 1: xargs + =--files-without-match= to get narrowed file set (temp file 1)
- Step 2: Re-run root pattern against narrowed set via second temp file
- Both steps produce grep-mode output
*** DONE Spawn child buffer
- Name: =*haystack:DEPTH:term1:term2:...*=
- Build descriptor by copying parent's and appending to =:filters=
- Set =haystack--parent-buffer= to current buffer

** Buffer Tree Navigation
*** DONE =haystack-go-up=
- Check =haystack--parent-buffer=, switch if live, no-op + message if nil/dead
*** DONE =haystack-kill-node=
- Kill just this buffer
*** DONE =haystack-kill-subtree=
- Recursively kill this buffer and all descendants
*** DONE =haystack-kill-whole-tree=
- Walk to root via parent chain, then kill subtree
*** DONE =haystack-kill-orphans=
- Kill buffers whose parent is dead AND have no living children
- Orphans with children become de facto roots — leave them

** MOC Generator
*** DONE =haystack-copy-moc= and =haystack-yank-moc=
- Deduplicate results by filename (first match per file)
- Format links based on file extension:
  - Org: =[[file:path::line][Pretty Title]]=
  - Markdown: =[Pretty Title](path#Lline)=
  - Plaintext: =Pretty Title — path:line=
  - Code files: comment-syntax link
- =haystack-copy-moc= stores loci; =haystack-yank-moc= inserts at point and pushes to kill ring

** Tree View & Header Navigation
*** DONE =haystack-show-tree=
- Dedicated =*haystack-tree*= buffer showing the full forest of open haystack buffers
- Indented by depth: root at col 0, each filter level indented further
- Current buffer marked with =←=
- Each line carries a text property pointing to its buffer
- =RET= visits the buffer at point; =q= closes the tree window
- Persists until closed — does not auto-close on navigation
- Multiple disconnected trees (separate root searches) separated by blank lines
- Derived from =special-mode= (read-only, =q= → =quit-window= for free)
- Per-item and sibling navigation (n/p, M-n/M-p); cursor lands on term text, not art
- Depth-level faces drawn from active theme via =font-lock-*= face list
*** DONE Children picker
- When navigating to children from a buffer with one child: go directly
- When multiple children exist: open =*haystack-children*= listing them
- Each line is the child's chain label; =RET= navigates to it
*** DONE Header navigation buttons
- Navigation line in results header: =[root]  [up]  [down]  [tree]=
- Implemented as =button= text properties — keyboard (=RET= on button) and mouse accessible
- =RET= in =haystack-results-mode= routes to =push-button= if on a button,
  otherwise falls through to =compile-goto-error=
- Buttons live entirely within the read-only header region
- =haystack-go-root= walks the parent chain to the root buffer

** Phase 1 Polish
*** DONE Keybinding maps
- =haystack-results-mode-map=: RET, n/p navigation, f filter, u/d go-up/down, k/K/M-k kill variants, c MOC, t tree, ? help
- =haystack-prefix-map=: s root search, r search region, n new note, y yank-moc, t show tree
- Prefix map unbound by default; user adds =(global-set-key (kbd "C-c h") haystack-prefix-map)=
*** DONE =?= help menu
- =*haystack-help*= buffer in =special-mode=, displayed below results window
- Key bindings read live from =haystack-results-mode-map= via =where-is-internal=
- Focus moves to help window on open; =q= closes and returns to results buffer
*** DONE Modifier prefixes in buffer and tree names
- Buffer names encode modifiers: =*haystack:2:rust:/cargo*=, =*haystack:2:rust:!/async*=
- Tree labels match: negation (=!=), filename (=/=), literal (==), regex (=~=) all shown
*** DONE README
- User guide covering setup, note creation, search, filtering, modifier prefixes,
  results buffer keys, tree view, MOC, and customization options

* Phase 2 — Retrieval Memory
Expansion groups and frecency. Force multipliers that make the most sense once a base corpus exists.

** Definition of Done
- [ ] Expansion groups load, validate, and expand search terms
- [ ] =haystack-associate= handles all three states correctly
- [ ] =haystack-rename-group-root= updates groups, frecency, composites atomically
- [ ] Exclusivity guardrail blocks redundant filters and suggests ~=~
- [ ] Frecency records every buffer, scores on read, replays leaf-only
- [ ] Deferred writes flush on idle timer and shutdown
- [ ] ~=~ literal prefix works in both root search and filter
- [ ] Buffer headers show expanded query feedback

** Expansion Groups — Data Layer
*** TODO Load =.expansion-groups.el= with =condition-case=
- On failure: warn, set empty alist, continue
*** TODO =haystack-validate-groups=
- Iterate all groups, collect terms in hash table, flag duplicates
- Emit warning buffer on conflicts
- Run on every load
*** TODO =haystack-describe-expansion-groups=
- Dump full alist in readable format

** Expansion Groups — Search Integration
*** TODO Expansion group lookup in input pipeline
- Single-word, no ~=~ prefix: look up group (case-insensitive)
- If found: construct =(escaped_A|escaped_B|escaped_C)= alternation
- Each member through =regexp-quote= before insertion
*** TODO Expanded query feedback in buffer header
- Show full expansion: =root=(Programming|Coding|Code|Scripting)=
*** TODO Exclusivity guardrail for filtering
- Block single-word filter if term already in root expansion (either direction)
- Suggest ~=~ prefix
- Multi-word phrases bypass this check entirely

** Expansion Groups — Management
*** TODO =haystack-associate= three-state flow
- State 1: B unassigned → prompt to add B to A's group or create new
- State 2: B in different group → fail, show both groups, offer move/abort
  - Move = detach single term, add to other group
  - If moved term was root, next member becomes root
  - If only member, dissolve group
- State 3: B already in A's group → no-op, inform
- Reject multi-word terms (whitespace check)
*** TODO =haystack-rename-group-root=
- Update groups file
- Update frecency entries
- Update composite =SOURCE-CHAIN= fields
- Rename composite files
- Atomic: all or warn
*** TODO =haystack-dissolve-group=
- Remove group entirely
- Downstream composites/frecency remain valid (lose expansion on replay)

** Frecency Engine — Core
*** TODO Data structure: =haystack-frecency-data=
- Alist in =.haystack-frecency.el=
- Entry: =((term1 term2 ...) :count N :last-access TIMESTAMP)=
- Terms preserve prefix characters (=!=, ~=~, =~=)
*** TODO Load frecency data with =condition-case=
*** TODO =haystack--frecency-record=
- Called on every buffer creation (root search and filter)
- Derive chain from =haystack--search-descriptor= (not buffer name)
- Increment count, update timestamp in memory
- Set =haystack--frecency-dirty= flag
*** TODO Scoring function
- =score = visit_count / max(days_since_last_access, 1)=
- Computed on read, never stored

** Frecency Engine — Persistence
*** TODO Deferred write system
- =haystack--frecency-dirty= boolean flag
- Idle timer (60 seconds) checks flag, writes if dirty
- =kill-emacs-hook= also flushes
- Never write on individual buffer creation
*** TODO =haystack-describe-frecent=
- Dump frecency list with computed scores for inspection

** Frecency Engine — Completion & Replay
*** TODO =haystack-frecent= via =completing-read=
- Present all recorded paths sorted by score descending
- Vertico/Orderless work immediately
*** TODO Replay: leaf buffer only
- Apply root expansion and each filter step internally
- Materialize only final result buffer
- Full =haystack--search-descriptor=, nil =haystack--parent-buffer=
- Buffer name reflects full chain
*** TODO All/Leaf toggle
- Transient menu switch between all atoms and leaf-only display
- Leaf = entries not a prefix of any higher-scored entry

** Composite Filter (Transient Menu)
*** TODO Three-state toggle: exclude / only / all
- =--glob '!@*'= to exclude (default)
- =--glob '@*'= to isolate
- No glob for all
- Child buffers inherit from parent descriptor, not transient state


* Phase 3 — Synthesis & Diagnostics
The most complex subsystems. Benefit from real usage data.

** Definition of Done
- [ ] =haystack-compose= creates read-only buffer from search results
- [ ] =C-c C-c= writes composite with correct slug, frontmatter, sentinel
- [ ] =C-c C-k= kills without saving
- [ ] One composite per canonical chain, overwrites on regeneration
- [ ] Composites surface in buffer headers regardless of filter state
- [ ] Discoverability command produces diagnostic buffer with reach/isolated terms
- [ ] Stop words load, seed, add, remove, describe all work

** Composite Notes
*** TODO Canonical chain normalization
- Resolve each term to expansion group root, lowercase
- =programming > rust > bevy= from either "coding" or "programming" input
*** TODO Check for existing composite
- =xargs -0 rg= scoped to =@*= files, match =SOURCE-CHAIN=
*** TODO =haystack-compose=
- Extract filenames from current buffer
- Read full file contents for each
- Build read-only buffer with delimiters and links
- Pre-populate from existing composite if found
*** TODO Composite buffer keybindings
- =C-c C-c=: write to =@comp__canonical-chain.ext=
  - Generate frontmatter (TITLE, LAST-GENERATED, SOURCE-CHAIN, SOURCE-COUNT)
  - Insert sentinel
  - Overwrite existing with =y-or-n-p= confirmation
- =C-c C-k=: kill buffer without saving
- Intercept =write-file= and =save-buffer=
*** TODO Composite slug convention
- =@comp__canonical-chain.ext=
- Extension from =haystack-composite-format= defcustom (default =org=)
*** TODO Composite surfacing in buffer headers
- On every root search and filter: one extra =rg= scoped to =@*= files
- Display =;;; haystack-composite: @comp__chain.ext= if found
- Machine-parseable, navigable link
- Runs regardless of composite filter state
*** TODO Composite format templates
- Org (default), Markdown, Plaintext — per spec Section 3F

** Discoverability Engine
