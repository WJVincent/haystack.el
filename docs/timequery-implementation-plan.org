** DONE Add insertion commands
Two commands with distinct behaviors:
- =haystack-insert-timestamp-now= inserts the current exact time with
  no prompt.
- =haystack-insert-timestamp= prompts for a date string.  Accepts
  =YYYY-MM-DD= (produces a date-only stamp) or =YYYY-MM-DD HH:MM=
  (produces a full timestamp).  No expansion or inference: the user's
  input precision determines the output format.  Does not use the
  query parser's range-expansion logic.

** DONE Implement timestamp extraction and date filtering
The inner loop uses float-time (seconds since epoch) as the canonical
comparable representation. Bounds are resolved once via
=encode-time= + =float-time=; per-line comparison is then a simple
float inequality.  To avoid inner-loop branching, pre-calculate two
sets of float bounds at search start: strict bounds for time-point
timestamps and day-expanded bounds (midnight to 23:59:59) for
date-only timestamps.  The inner loop selects the bound set based on
timestamp type and performs a pure float comparison.

** DONE Refactor frecency key to descriptor-shaped structure
The current chain key is a list of prefixed strings — a serialization
that loses structure and requires reparsing at replay time.  Replace
it with a descriptor-shaped plist key for all root types so replay
dispatch is structural, not string-parsing.

Text root key shape: =(:root (:kind text :term "rust") :filters
((:term "async") ...)=

Date root key shape: =(:root (:kind date-range :start "2024-01" :end
"2024-03") :filters ())=

=assoc= continues to work since it uses =equal= for structural
comparison.  Replay dispatch becomes =(plist-get root :kind)= with no
special-casing.
